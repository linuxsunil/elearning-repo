## 🎨 Design Methodologies

### Primary Framework: ADDIE Hybrid (ADDIE + SAM)

I follow a **structured yet agile approach** that combines:

**ADDIE** (Analysis → Design → Development → Implementation → Evaluation)
- Rigorous upfront analysis & learning objectives
- Systematic curriculum design
- Quality assurance checkpoints

**SAM** (Successive Approximation Model)
- Iterative prototyping with stakeholder feedback
- Rapid course revisions based on pilot data
- Continuous improvement cycles

**Result**: Predictable timelines + flexible adaptation to feedback

## 📡 Technical Standards & Compliance

### SCORM Integration
- **SCORM 1.2** — Legacy LMS compatibility, CMI data model
- **SCORM 2004 (3rd/4th edition)** — Modern standard, completion/success patterns
- **cmi5** — xAPI profile for LMS-launched content
- **Expertise**: Manifest structure, score/completion rollup, data persistence, imsmanifest validation

### xAPI (Experience API)
- **Actor modeling** — Learner identification (name, email, LRS endpoint)
- **Verb registries** — ADL vocab URIs (completed, attempted, answered, passed, failed, interacted, experienced)
- **Activity definitions** — Course, module, interaction, assessment, interaction types
- **Result objects** — Score (raw, min, max, scaled), duration (ISO 8601), completion status
- **Context extensions** — Custom properties for industry-specific tracking

### Accessibility (WCAG 2.1 Level AA)
- **Images**: Descriptive alt text (function, not appearance), text alternatives for charts
- **Video**: Closed captions (SRT/VTT), audio descriptions where visual content carries meaning
- **Colour & Contrast**: 4.5:1 minimum (normal text), 3:1 (large text ≥18pt or 14pt bold)
- **Interactions**: Keyboard navigability, visible focus indicators, no timed interactions without controls
- **Reading Level**: Narration ≤ Flesch-Kincaid Grade 10, on-screen text ≤ Grade 8

### LMS Targets
- **Docbo**, **Moodle**, **Cornerstone**, generic SCORM-compliant systems
- **Expertise**: Package import/export, data integration points, report bridging

## 📊 Quality Assurance & Review Protocol

I follow a **7-point QA framework** (see CLAUDE.md Section 7 for full checklist):

### 1. **ID Quality Review**
- [ ] Each slide maps to a learning objective (LO code present)
- [ ] All objectives use Bloom's revised verbs at correct cognitive level
- [ ] Assessment items align to stated learning objectives
- [ ] Feedback text present for all assessment items (correct & incorrect paths)
- [ ] Reading level meets targets (narration ≤ Grade 10, on-screen ≤ Grade 8)

### 2. **xAPI Statement Review**
- [ ] Verb IDs use approved ADL vocab URIs
- [ ] Actor object contains objectType, name, mbox
- [ ] Activity IDs follow standard pattern
- [ ] Result scores scaled 0.0–1.0
- [ ] Duration in ISO 8601 format

### 3. **SCORM Compliance Review**
- [ ] Correct CMI fields for target SCORM version
- [ ] Completion/success status logic handles all states
- [ ] masteryScore matches pass mark
- [ ] imsmanifest identifiers unique
- [ ] No broken href references

### 4. **Accessibility Review (WCAG 2.1 AA)**
- [ ] All images have alt text (or alt="" if decorative)
- [ ] Video has caption file
- [ ] No color-only information encoding
- [ ] No timed interactions without learner controls
- [ ] Tab order matches visual reading order
- [ ] Contrast ratios ≥4.5:1 (normal text) or ≥3:1 (large text)

### 5. **Content Accuracy Review**
- [ ] Technical claims verified against sources
- [ ] Statistics & dates cited with footnotes
- [ ] No placeholder text (Lorem Ipsum, TBD)
- [ ] Brand names match client style guide

### 6. **File & Naming Convention Review**
- [ ] All files use kebab-case
- [ ] Follow {course-slug}-{type}-{descriptor} pattern
- [ ] Storyboard slides use required Markdown template
- [ ] LO codes in LO-{module}-{number} format

### 7. **Learner Experience Review**
- [ ] Engagement level appropriate for audience
- [ ] Interactions support learning (not just decoration)
- [ ] Feedback is timely & constructive
- [ ] Navigation is intuitive
- [ ] Course flow supports retention

## 📚 Standards & Best Practices

### British English by Default
- Spelling: colour, organisation, realise (not color, organization, realize)
- Exceptions: Client spec or US-based content only

### Learning Objective Format
```
LO-{module}-{number}: Learners will be able to {Bloom's verb} 
{knowledge/skill} {condition/context}.

Example:
LO-02-03: Learners will be able to distinguish between supervised 
and unsupervised learning given a real-world dataset description.
```

### Assessment Item Format
All assessment items must include:
- Clear question stem (no ambiguity)
- Plausible distractors (common misconceptions)
- Correct answer with justification
- Feedback for each option (correct & incorrect paths)
- Mapping to learning objective

### Narration Script Guidelines
- Max 20 words per sentence (supports audio pacing)
- Conversational tone (avoid dense academic language)
- Define acronyms on first use in each module
- Grade level: Flesch-Kincaid 10 or below
